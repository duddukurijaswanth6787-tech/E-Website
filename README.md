# 👗 Vasanthi Creations: Luxury Boutique ERP & Storefront
### Enterprise-Grade Operational Control System for Bespoke Production

Vasanthi Creations ERP is a specialized, end-to-end operational platform designed to bridge the gap between luxury fashion retail and workshop production floor execution.

---

## 🌟 Core Strategic Value

This platform transforms traditional boutique operations into a digitally-orchestrated production ecosystem:

*   **Showroom Excellence**: Luxury storefront with an industrial-grade bespoke measurement engine.
*   **Operational Intelligence**: Live Kanban-based production tracking with real-time bottleneck detection.
*   **Workshop Coordination**: Direct "Manager-to-Tailor" briefing system with instant status synchronization.
*   **Executive Oversight**: Data-driven insights into revenue, production throughput, and SLA adherence.

---

## 🛠 Technology Stack

The architecture is built for high-concurrency production environments:

*   **Frontend**: React (Vite) + Tailwind CSS (v4) + Framer Motion (Luxury UI/UX).
*   **Backend**: Node.js (TypeScript) + Express.
*   **Real-time Engine**: Socket.IO for cross-portal event synchronization.
*   **Database**: MongoDB (Domain-driven modeling).
*   **Caching/OTP**: Redis.
*   **Deployment**: Docker-ready with production security hardening.

---

## 📁 Key Ecosystem Portals

### 🏢 Super Admin Portal
The command hub for overall business administration. Includes product management, customer CRM, audit logs, and global analytics.

### 💼 Manager Operations Portal
The heartbeat of the production floor. Features a real-time Kanban board, tailor workload heatmaps, and a direct communication bridge to tailors.

### ✂️ Tailor Production Portal
A mobile-optimized, high-focus interface for production personnel to receive briefs, view measurements, and update task status instantly.

### 🛍 Customer Storefront
A premium e-commerce experience with deep customization flows for bespoke sarees and blouses.

---

## 🚀 Quick Start (Development)

1.  **Dependencies**: Run `npm install` in both `/backend` and `/Frontend`.
2.  **Environment**: Configure `.env` files using the provided `.env.example` templates.
3.  **Boot**: Run `npm run dev` in both directories.
4.  **Access**: Visit `http://localhost:5173` for the storefront and `/admin` for management.
