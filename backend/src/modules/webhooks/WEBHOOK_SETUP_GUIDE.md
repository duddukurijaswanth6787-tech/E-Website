# Comprehensive Razorpay Webhook Configuration & Testing Guide

This guide outlines exactly how to test, configure, and secure the **Razorpay Webhook Processing engine** implemented within your MERN back-office infrastructure.

---

## 1. Local Testing Setup (ngrok Tunnelling)

Since external Gateway servers like Razorpay cannot ping local development environments (`localhost`), you must establish a secure external proxy tunnel using **ngrok**.

### Execution Steps:
1. Start your local Express development target instance:
   ```bash
   cd backend && npm run dev
   ```
2. Open a separate terminal and start ngrok targeting your backend API port (e.g., `5000`):
   ```bash
   ngrok http 5000
   ```
3. Copy the secure `https` Forwarding URL generated by ngrok (e.g., `https://a1b2c3d4.ngrok-free.app`).

---

## 2. Live Razorpay Dashboard Mapping

Register your live endpoint to enable automated transactional hooks.

### Configuration Instructions:
1. Navigate to the **Razorpay Dashboard** → **Settings** → **Webhooks**.
2. Click **Add New Webhook**.
3. **Webhook URL**: Enter your full public gateway path:
   ```text
   https://YOUR_DOMAIN/api/payments/razorpay/webhook
   ```
   *(For local testing, paste your ngrok URL: `https://a1b2c3d4.ngrok-free.app/api/payments/razorpay/webhook`)*
4. **Secret**: Enter your high-entropy string key matching `RAZORPAY_WEBHOOK_SECRET` inside your `.env` target:
   ```text
   your_webhook_secret
   ```
5. **Active Events Checklist**: Select exactly the following operations:
   - `[x]` `payment.authorized`
   - `[x]` `payment.captured`
   - `[x]` `payment.failed`
   - `[x]` `order.paid`
   - `[x]` `refund.processed`
6. Click **Create Webhook** to complete initial registration handshakes.

## 3. Signature Generation Helper Script (For Standalone Testing)

To emulate cryptographic digest handshakes manually without live Gateway clusters, use this inline Node.js script.

### Create `generateMockSig.js`:
```javascript
const crypto = require('crypto');

const secret = 'your_webhook_secret';
const payload = JSON.stringify({
  event: 'payment.captured',
  account_id: 'acc_test_123',
  created_at: Date.now(),
  payload: {
    payment: {
      entity: {
        id: 'pay_test_999',
        order_id: 'order_test_888',
        amount: 250000, // ₹2,500.00
        currency: 'INR',
        status: 'captured'
      }
    }
  }
});

const signature = crypto
  .createHmac('sha256', secret)
  .update(payload)
  .digest('hex');

console.log('--- Mock JSON Payload Header ---');
console.log(payload);
console.log('\n--- Cryptographic Signature ---');
console.log(signature);
```

## 4. Postman Verification Test Sequence

Use Postman to test raw buffer extraction and verify idempotency.

### Setup Instructions:
1. Open **Postman** and create a new **POST** request targeting:
   ```text
   http://localhost:5000/api/payments/razorpay/webhook
   ```
2. Navigate to the **Headers** panel and add:
   - **Key**: `x-razorpay-signature`
   - **Value**: Paste the exact HMAC signature generated by your helper script.
   - **Key**: `Content-Type`
   - **Value**: `application/json`
3. Navigate to the **Body** panel, select **raw**, and paste the exact corresponding JSON string.
4. Click **Send**.
5. **Expected Output (First Hit)**:
   ```json
   {
     "success": true,
     "message": "Webhook stream lifecycle completely processed."
   }
   ```
6. **Expected Output (Subsequent Hits - Idempotency Validation)**:
   ```json
   {
     "success": true,
     "message": "Idempotency checkpoint verified: Duplicate execution skipped."
   }
   ```
